# 个人研究辅助路线图

本文档定义项目下一阶段方向：把 topkyo AI 基建研究台定位为个人研究辅助工具，用于发现研究线索、暴露数据质量和沉淀复盘记录；不作为交易系统、自动下单系统或投资建议生成器。

## 定位边界

- 输出“研究优先级、证据链、风险提示、跟踪结果”，而不是可直接执行的交易指令。
- `buy` / `hold` / `sell` 仅作为辅助标签，后续应弱化为研究状态或关注等级。
- 不接券商接口，不做自动下单，不同步真实账户持仓，不提供一键交易能力。
- 回测和历史表现只用于校准研究流程，不用于宣传收益或证明未来有效性。
- 人工复核是最终决策入口，系统只负责减少漏看、少踩数据坑、提升复盘一致性。

## P0：必须先解决

### 1. 信号证据链

当前 LLM 理由过于概括，后续每个研究建议都必须绑定可审计输入：

- 估值：PE(TTM)、PB、利润增速、PEG 或缺失原因。
- 动量：20 日涨跌幅、趋势状态、异常涨跌幅。
- 主题：所属环节、是否全球供应链、主题分层。
- 数据质量：字段来源、缓存/实时状态、warning、unavailable。
- 结论依据：LLM 使用了哪些关键字段，而不是只给一句叙事。

验收标准：任一重点标的都能回答“为什么进入今日研究列表、哪些数据支持、哪些数据不可靠”。

### 2. 负面信息层

研究辅助不能只强化看多叙事。需要增加负面与排除维度：

- 减持、解禁、问询函、监管处罚、业绩预告不及预期。
- 商誉、应收账款、存货、现金流等财务质量风险。
- 订单证伪、客户集中度、海外限制、供应链不确定性。
- 估值极端、短期拥挤、放量滞涨、连续大涨后的回撤风险。

验收标准：每个重点候选至少显示“主要支持因素”和“主要反证/风险因素”。

### 3. 信号收敛

最近快照中 `buy` 数量偏多，作为研究辅助会稀释注意力。后续输出应改为分层：

- `重点研究`：今日最值得深入看的 5-10 只。
- `观察`：有主题或动量线索，但证据不足。
- `风险预警`：基本面、事件或数据质量存在明显问题。
- `排除`：短期不进入研究列表，并保留原因。

验收标准：首页或信号页优先呈现少量高优先级标的，而不是大面积 `buy` 列表。

### 4. 历史展示防误导

即使不用于交易，历史回看也会影响判断。当前回测仍存在前视偏差风险：

- 历史调仓日可能使用当前基本面。
- 当前股票池回放历史会产生幸存者偏差。
- LLM 可能带有调仓日之后的信息，`as_of` prompt 不能完全隔离。

验收标准：所有历史表现区域显式标注用途和局限；在点时数据未完成前，不把回测收益作为策略有效性证据。

### 5. LLM 稳定性评估

同一批输入在不同模型、批次顺序或缓存状态下可能产生不同结论。需要记录：

- 同一数据重复运行的一致率。
- 同一标的建议等级和置信度波动。
- 不同模型输出差异。
- LLM 返回 live/cache 的来源状态。

验收标准：重点标的能显示“连续强化、首次出现、结论反转、模型分歧”等状态。

## P1：高价值改进

- 研究优先级评分：综合证据强度、风险项、数据质量和变化幅度，回答“今天先看谁”。
- 主题拥挤度：板块内同涨比例、短期涨幅、成交额放大、涨停/跌停数量。
- 事件日历：财报、业绩预告、解禁、股东大会、重大公告。
- 股票池分层：核心龙头、弹性标的、周期修复、主题投机、防御底仓。
- 快照后跟踪：记录信号后 5/10/20 个交易日表现，只用于复盘研究质量。
- 变化榜：从观察升为重点、从重点降级、风险项新增、数据质量变差。

## P2：研究工作台体验

- 增加人工状态：待看、已读、重点跟踪、暂时排除、需复核。
- 增加研究笔记：每只股票保存人工结论、反证、下一次检查点。
- 增加结论历史：保留系统建议和人工判断的时间线，避免被每日新叙事带走。
- 首页突出异常：数据过期、非实时价、字段缺失、LLM 缓存、风险 warning。
- `/signals` 从交易信号页转为研究工作台：Top candidates、风险票、变化最大、连续强化。

## 暂不做

- 不接券商下单、不做自动交易、不处理真实资金。
- 不新增“收益最大化”自动调参目标。
- 不把未经点时修正的回测收益展示为策略能力。
- 不增加业务兜底：LLM、API 或关键数据失败时仍保持显式失败。
- 不用规则或缓存合成投资结论来掩盖失败。

## 外部边界参考

- SEC / Investor.gov 提醒：回测表现是 hypothetical，不代表实际表现，历史表现也不能预测未来。
- FINRA 提醒：自动化投资工具和 AI 投资内容不能替代投资者对自身目标、风险承受能力和信息来源可靠性的判断。
- 上交所程序化交易规则说明：若未来进入自动交易范畴，需要另行建设报告、风控、异常监测、错误处理和人工干预机制；当前项目不进入该范围。
